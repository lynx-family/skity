# AI Agent Unified Test Feedback Loop Design

## 1. Background and Motivation

For an AI Agent, extracting actionable context from raw terminal logs (like C++ compiler logs, GTest outputs, and raw image diff scripts) is extremely difficult. It causes the agent to waste tokens and time "guessing" the failure reasons. 

This document outlines the unified, machine-friendly test feedback loop designed to help the Agent efficiently debug both logical errors (Unit Tests) and visual regressions (Golden Tests) within the Skity project.

## 2. Unified Failure Report Schema

The core of this design is a structured JSON report generated by a unified Test Runner.

### 2.1 JSON Structure Example

```json
{
  "summary": {
    "total": 120,
    "passed": 118,
    "failed": 2,
    "duration_ms": 4500
  },
  "failures": [
    {
      "suite": "unit",
      "case_name": "MathTest.MatrixMultiplication",
      "status": "FAIL",
      "exit_code": 1,
      "stage": "Layer 1: Semantic Validation",
      "reason_code": "assert_failed",
      "backend": "cpu",
      "oracle": "assertion",
      "error": "Expected equality of these values: a and b",
      "artifacts": {
        "file": "src/math/matrix_test.cc",
        "line": 42
      }
    },
    {
      "suite": "golden-shape",
      "case_name": "SimpleShapeGolden.DrawFilledRect",
      "status": "FAIL",
      "exit_code": 1,
      "stage": "Layer 3: Visual Regression",
      "reason_code": "pixel_mismatch",
      "backend": "metal",
      "oracle": "golden",
      "metrics": {
        "diff_percent": 0.0015,
        "diff_pixel_count": 1000,
        "max_diff_percent": 1,
        "diff_bbox": [50, 100, 50, 10]
      },
      "artifacts": {
        "expected_image": "test/golden/cases/simple_shape/simple_images/draw_filled_rect.png",
        "actual_image": "test/golden/cases/simple_shape/simple_images/draw_filled_rect_actual.png",
        "diff_pixels_file": "test/golden/cases/simple_shape/simple_images/draw_filled_rect_diff_pixels.json",
        "file": "test/golden/cases/simple_shape/simple_shape.cc",
        "line": 66
      }
    }
  ]
}
```

### 2.2 Key Fields (Agent Perspective)

- **`stage` & `reason_code`**: Instructs the Agent on what type of error occurred (`assert_failed` vs `pixel_mismatch`).
- **`metrics.diff_bbox`**: A bounding box `[x, y, width, height]` of all mismatched pixels. It helps the Agent logically locate where the rendering went wrong on the canvas without relying solely on visual multi-modal capabilities.
- **`artifacts.diff_pixels_file`**: Contains a precise JSON array of the mismatched pixels (up to 1000 pixels), detailing the exact `src` (actual rendering) and `dst` (golden target) RGBA channel differences.

## 3. Implementation Details

1. **Test Runner (`tools/test-runner.py`)**: 
   - The single entry point for all tests. It orchestrates the underlying C++ binaries, captures crashes, and parses GTest JSON outputs into the unified schema.
   - Outputs the final report wrapped in `<agent-test-report>` XML tags to the console and persists it to `build/agent_test_report.json` in the root directory (or whatever `--build-dir` is configured to).
   
2. **Golden Test Pixel Diffing (`test/golden/common/golden_test_check.cc`)**:
   - Upon a pixel mismatch, it automatically saves the `actual_image` and generates the `diff_pixels_file`.
   - Computes the `diff_bbox` during the pixel comparison loop and prints these metrics to `stdout` under a `[[GOLDEN_TEST_FAILED_METRICS]]` block, which the Python runner then intercepts and merges into the final report.

## 4. Agent Workflow (Enforced)

As defined in `AGENTS.md`, the AI Agent **MUST**:
- Use `python3 tools/test-runner.py --suite=<suite>` to run tests.
- Read the resulting `build/agent_test_report.json` to understand the exact failures.
- If a `pixel_mismatch` occurs, the Agent **MUST** read the `diff_pixels_file` (using the `Read` tool) to programmatically analyze the rendering discrepancy.
